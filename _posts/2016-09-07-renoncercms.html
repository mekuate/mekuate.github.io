Un Sytème de gestion des contenus (en anglais CMS, Content Management System) permet de créer des sites web ayant un contenu dynamique. Les CMS gèrent les workflow de contenus, la persistence des données dans des bases de données.

les CMS les plus utilisés sont Wordpress, Drupal et Joomla.

Bien que les CMS peuvent réaliser des sites web des plus simples aux plus complexes, il est tout de même important de prendre d'identifier très tôt dans les projets de création de site web s'il faut utiliser un CMS ou non.
Cet article permet de répondre à cette problématique

Quand utiliser un CMS

Pour l’une des raisons suivantes, nous devons utiliser un CMS:

  • Des sites web dynamique et très actif en contenu qui nécessite une catégorisation, une typologie de contenus;
  • des sites web avec plusieurs rédacteurs , et qui nécessite un workflow de gestion de contenus très important.

Quand renoncer à utiliser un CMS

Pour des raisons suivantes, nous devons renoncer à utiliser un CMS et créer à la place un site web statique
  • Des sites web dits “brochures” ne nécessite pas l’usage de CMS, car ces sites disposent d’informations qui ne changent pas generalement, voir trop peu. par exemple, un site web de présentation d’une entreprise n’a pas réellement besoin d’être fait avec Drupal, wordpress ou Joomla lorsqu’il suffit juste de quelques pages HTML, une dose de feuille de style CSS pour présenter toute l’entreprise.
  • Des Blogs peuvent très bien être déployé avec un site web statique, simpla avec le focus sur le plus important: le contenu

Renoncer à utiliser un CMS, revient à utiliser un générateur de site web statique comme Jekyll pour faciliter la mise en oeuvre du site web statique.

D'ailleurs ce site web est construit entièrement avec Jekyll.

Conclusion

Avant de se lancer dans la création d’un site web, bien qu'un CMS soit capable de tout faire, il faut penser aussi si c’est nécessaire car les CMS sont gourmands en ressources contrairement à des sites statiques très léger (pas de bases de données) et facile à mettre en oeuvre (technologie web de base).
